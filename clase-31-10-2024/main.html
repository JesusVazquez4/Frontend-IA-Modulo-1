Proceso para crear un repositorio en GitHub

Para ver la versión de git instalada y con ello confirmar que quedó instalado bien usamos el comando.

        git -v

También con el comando

        git --version

Configurando git

Para configurar el nombre de usuario de git en la terminal usamos

        git config --global user.name "Nombre de usuario"

Para configurar el correo de usuario de git en la terminal usamos

        git config --global user.email "Correo de usuario"

Esto solo se ejecuta una vez por sesión.

Para ver la configuración de git en la terminal usamos

git config --list

Es importante corroborar que apareca el nombre de usuario y el correo. Tenemos que dar enter hasta que se muestre el listado completo.

Crear un repositorio en GitHub

Ahora creamos un repositorio en GitHub. Y compiamos el enlace al repositorio, el que está en HTTPS.

El enlace al repositorio es: https://github.com/JesusVazquez4/Frontend-IA-Modulo-1.git


Para lo que sigue debemos de tener cuidado de que al usar la terminal que estemos usando, estar en la carpeta del proyeto que vamos a subir en el repositorio en GitHub.

Subir al repositorio

Empaquetado

Primero preparamos lo que vamos a enviar al repositorio, para eso le decimos primero a Git que vamos a estar trabajando en la carpeta actual, que es la carperta que contiene los archivos a subir. Para ello usamos el siguiente comando, que hará un repositorio vació:

        git init

Este comando, de git init, solo se hace una por proyecto.

Ahora para "empaquetar" lo que vamos a subir, usamos el siguiente comando:

        git add .

En el caso de que solo queramos subir un archivo usamos el siguiente comando:

        git add archivo.txt

Por ejemplo, aquí solo empaquetamos el archivo "archivo.txt".

Ahora para ver cuales son los cambios que queremos enviar usamos el comando:

        git status

En el caso de que solo queramos ver los cambios de un solo archivo es con el siguiente comando:

        git status archivo.txt

Commit

Ahora vamos a hacer un commit, para eso usamos el siguiente comando:

        git commit -m "Comentario"

En el caso de que solo queramos hacer un commit de un archivo es con el siguiente comando:

        git commit -m "Comentario" archivo.txt

En el caso de que queramos hacer un commit de todos los archivos es con el siguiente comando:

        git commit -m "Comentario" *

La parte del comentario es importante se específico para saber que clase de cambio se hizo.

Podemos verificar dando de nuevo el comando git status.

        git status

Con esto terminamos de registrar los cambios, pero aún no se envía nada.

Enlazamos comunicación con GitHub

Conectamos con nuestro repositorio de GitHub con el enlace que hemos copiado y el siguiente comando:

        git remote add origin https://github.com/JesusVazquez4/Frontend-IA-Modulo-1.git

Envío

Para que se envie el proyecto a GitHub, usamos el siguiente comando:

        git push -u origin main

Y en algunos casos el origen esta ubicado en master y esta en master, en cuyo caso se usa:

        git push -u origin master

Esto esta relacionado con las ramas, lo que checamo en que rama esta activa con el siguiente comando:

        git branch

En este caso, la rama activa es main.

Si queremos cambiar la rama activa, usamos el siguiente comando:

        git checkout -b nombre_rama

Por ejemplo, para cambiar la rama activa a master, usamos el comando:

        git checkout -b master

Podemos checar los cambio directo en GitHub y en terminal podemos hacerlo con el comando:

        git log
